Hexo 搭建个人博客

Yuanhy

2021-09-12 16:14:19

本篇文章主要说明关于利用现有的 Hexo 搭建个人博客，Hexo 的下载安装以及启动服务访问

为什么使用 Hexo 搭建个人博客

以下仅代表个人想法，欢迎评论

  • 网上广告太多
    • 当我在网上浏览技术文章时，时不时弹出广告遮盖内容，而我的目的仅仅是为了阅读，当广告遮住了内容，我需要额外花费时间处理这个问题，因此，我需要一个干净没有广告的阅读环境，而自定义个人博客符合我的想法
  • 记录生活
    • 在工作生活中，因为一段时间内经历的事情太多，有时候会忘记事情，好记性不如烂笔头，这里将记录一些技术文章和生活的点点滴滴，方便在工作或其他时候可以温故而知新
  • 装逼
    • 当有一天，别人可以访问我的博客并且有收获时，自我感觉良好，很自豪，装的一手好逼(表情:狗头)
  • Hexo 的优势和我自身的原因
    • Hexo 是一个快速、简单、强大的博客框架，快速构建，支持 markdown 语法，支持一键部署，插件多，主题多，主要是本人为 Java 后端开发，了解前端 html ,css,js 基本使用，足以处理 Hexo 基本使用，而其他的博客框架像 wordpress(php 语言) 或者 Jekyll(Ruby 语言)，这些语言都没了解，对于我来说，时间有限，目前只是希望可以快速搭建，因此只考虑 Hexo

下载安装 Hexo

当前的环境要求

  • 安装 Hexo 非常简单，只需要事先准备好以下几点：

    • Node.js (Node环境至少 Node.js 10.13, recommends 12.0 or higher)

    • Git

      如果以上环境都安装配置好了，可以直接忽略以下安装 Git 和 安装 Node , 跳到安装 Hexo 部分

  • 安装 Git

    • Windows：下载并安装git。

    • Mac：使用Homebrew、MacPorts或installer 安装。

      • 对于 Mac 用户

        编译时可能会遇到一些问题。请先从 App Store 安装 Xcode。Xcode安装完成后，打开Xcode，进入Preferences -> Download -> Command Line Tools -> Install安装命令行工具。

    • Linux（Ubuntu、Debian）： sudo apt-get install git-core

    • Linux（Fedora、Red Hat、CentOS）： sudo yum install git-core

  • 安装 Node

    Node.js为大多数平台提供官方安装程序。

    其他安装方法：

    • Windows：使用nvs（推荐）或nvm 安装。
    • Mac：使用Homebrew或MacPorts安装。
    • Linux（基于 DEB/RPM）：使用NodeSource安装它。
    • 其他：通过相应的包管理器安装。请参阅Node.js 提供的指南。

安装 Hexo

  • 打开 Git 的 bash 环境

    上面我们已经安装了 Git ，那么 Git 安装目录下有一个 git-bash.exe,双击打开（git 可以配置在当前目录下下打开，没必要到安装木下打开，关于 Git 的使用自行了解）

  • 在磁盘中创建一个文件夹，以我的为例,D 盘下的data 目录下创建一个 fluidBlog ,因为等一下使用 fluid 主题，所以以这个命名文件夹，名字自定义无限制,如下所示

  • 在 fluidBlog目录下执行命令安装 Hexo : npm install -g hexo-cli

  • 安装 Hexo 后，运行以下命令在 fluidBlog 中初始化 Hexo 文件夹

    $ hexo init <文件夹>
    $ cd <文件夹>
    $ npm install

    注意：在这里显示失败，Please run 'npm install' in "D:\data\fluidBlog\blog" folder.需要进去 blog 文件夹执行 npm install

    但还是报错了，无法安装 ejs@2.7.4(因为我的环境被搞得乱七八糟，暂时找不到原因，也有可能是因为镜像是外国的，有点慢的原因，导致失败)

    解决如下

    选用 cnpm 和使用淘宝镜像

    • 查看注册的镜像，只有官方的镜像

    • 配置淘宝镜像

    • 安装 cnpm 命令

  • 在 blog 目录下执行 cnpm install

    即使报错，也没关系

本地部署

  • 部署前生成静态文件，命令 ：hexo g
  • 启动服务:http://localhost:4000 ，命令：hexo s
  • 在浏览器打开 http://localhost:4000 ,可直接访问 Hexo 原始主题的博客
